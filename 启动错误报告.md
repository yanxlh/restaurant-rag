# 启动错误排查报告

**项目：** restaurant-rag  
**日期：** 2026-05-10

## 错误一：MySQL 端口冲突

**现象**
```
Error response from daemon: ports are not available: exposing port TCP 0.0.0.0:3306
bind: address already in use
```

**原因**  
本地已有 MySQL 实例在运行，Docker 无法再绑定 3306 端口。

**解决方案**  
放弃用 Docker 启动 MySQL，直接使用本地已有的 MySQL 实例。启动时只运行基础设施中的 ChromaDB 和 Redis：
```bash
docker-compose up chromadb redis -d
```

---

## 错误二：Maven Wrapper 缺失

**现象**
```
zsh: no such file or directory: ./mvnw
```

**原因**  
项目未提交 Maven Wrapper 文件（`mvnw`、`.mvn/` 目录）。

**解决方案**  
使用系统全局安装的 Maven 直接运行：
```bash
mvn spring-boot:run
```

## 错误三：ChromaDB 版本与 Spring AI API 不兼容（核心问题）

**现象**
```
BeanCreationException: Collection [restaurant_reviews] does not exist
400 Bad Request: {"error":"InvalidCollection","message":"Collection restaurant_reviews does not exist."}
```

**原因（通过反编译 spring-ai-chroma-store-1.0.0.jar 定位）**

Spring AI 1.0.0 使用 ChromaDB **v2 API**，调用路径为：
```
GET /api/v2/tenants/{tenant}/databases/{database}/collections/{collection}
```
默认租户为 `SpringAiTenant`，默认数据库为 `SpringAiDatabase`。

| ChromaDB 版本 | 问题 |
|---|---|
| `latest`（1.0.0）| v1 API 已移除，v2 路径结构不同，全部报错 |
| `0.5.23` | 只有 v1 API，不支持 `/api/v2/tenants/...` 路径 |
| `0.6.3` | 支持 v2 API，但集合不存在时报 400，错误消息格式与 Spring AI 期望的字符串 `"Collection [xxx] does not exists"` 不匹配，导致 `initialize-schema: true` 的初始化逻辑无法触发 |

**`initialize-schema: true` 为何失效**  
`ChromaVectorStore.afterPropertiesSet()` 先调用 `getCollection`，`getCollection` 内部只有当错误消息**精确等于** `"Collection [xxx] does not exists"` 时才返回 `null`（触发自动建表），否则直接抛出 `RuntimeException`。ChromaDB 0.6.3 的实际错误消息为 `"Collection xxx does not exist."`，格式不匹配，抛出异常，初始化中断。

**解决方案**  
使用 ChromaDB `0.6.3`，在启动后端前手动通过 v2 API 预创建租户、数据库和集合：
```bash
# 创建租户
curl -X POST http://localhost:8000/api/v2/tenants \
  -H "Content-Type: application/json" \
  -d '{"name": "SpringAiTenant"}'

# 创建数据库
curl -X POST http://localhost:8000/api/v2/tenants/SpringAiTenant/databases \
  -H "Content-Type: application/json" \
  -d '{"name": "SpringAiDatabase"}'

# 创建集合
curl -X POST "http://localhost:8000/api/v2/tenants/SpringAiTenant/databases/SpringAiDatabase/collections" \
  -H "Content-Type: application/json" \
  -d '{"name": "restaurant_reviews", "metadata": {"hnsw:space": "cosine"}}'
```

**docker-compose.yml 修改**  
将 `chromadb/chroma:latest` 改为 `chromadb/chroma:0.6.3` 以固定版本。
